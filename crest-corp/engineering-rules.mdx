All Crest engineering work follows our **25 Engineering Rules** covering source code handling, secrets, access, AI usage, and contractor conduct. The canonical version lives in the Get Started tab and is the single source of truth.

<Note>
This page intentionally does not duplicate the rule list. Duplicating it would create a drift risk: an edit to one copy would not update the other, and contractors would end up reading a stale version. Read the canonical page instead.
</Note>

## Why this page exists in Crest Corp

The Engineering Rules apply to every Crest worker, not just engineers writing code in `crestpetsystem`. Designers, ops, AI researchers, and partners doing project work all touch the same access patterns and the same confidential information. Putting a pointer here in the **Crest Corp** tab makes sure non-engineering audiences find the rules during their onboarding flow rather than hunting through a codebase-focused tab.
