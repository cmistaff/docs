## Problem Statement

Marcia, our AI veterinary assistant, responds to veterinarians and pet owners. Historically, quality review was manual and unscalable. The Judge introduces automated, repeatable scoring across multiple layers, capturing granular diagnostics and a composite score to accelerate iteration and safely unlock training on verified, high-quality responses.

## Three-Layer Grading System

<Tip>
**Analogy:** Think of three reviewers checking different aspects: format, process/tool use, and expert content quality.
</Tip>

### Layer 1: Structural Proofreader (Fast, Deterministic, Zero Cost)

**Primary question:** Is the response structurally correct?

- Empty/malformed detection
- Format conformance (e.g., required JSON fields and types: petName, recommendation, urgency)

**Score:** 0.0-1.0 | **Speed:** Milliseconds | **Cost:** Free. Instantly catches obvious failures.

### Layer 2: Tool Inspector (Fast, Deterministic, Zero Cost)

**Primary question:** Did Marcia use the correct tools in the correct order?

- Tool recall and precision (F1 score)
- Sequence correctness via Longest Common Subsequence (LCS)

**Example:** Expected: `[Lookup Pet] -> [Check Schedule] -> [Book Appointment]`; Actual: `[Check Schedule] -> [Lookup Pet] -> [Book Appointment]`. Set match is perfect but order penalty applies.

**Score:** 0.0-1.0 | **Speed:** Milliseconds | **Cost:** Free.

### Layer 3: Expert AI Reviewer (Slower, Uses Reasoning Model)

**Primary question:** Is the content factually correct, professional, and helpful?

- Factual accuracy vs. ground truth
- Tool usage quality and decision rationale
- Response quality (clarity, completeness, tone)
- Overall holistic judgment

**Score:** 0.0-1.0 | **Runs:** Nightly and on manual trigger | **Cost:** API per call (bounded by sampling/eval scope).

## Composite Confidence Score

The three layers are combined into a single composite confidence score (0.0-1.0).

| Layer | Weight (Full Run) | Weight (Fast Run) |
| --- | --- | --- |
| Layer 1 Structural Check | 15% | 30% |
| Layer 2 Tool Correctness | 35% | 70% |
| Layer 3 AI Quality Review | 50% | - |

When Layer 3 is skipped (e.g., PR checks), Layers 1 and 2 are re-weighted proportionally as shown.

## Storage and Usage of Scores

All evaluations are persisted to the `MarciaTrainingData` table for analytics and training curation.

| Field | Description |
| --- | --- |
| `confidenceScore` | Final composite score (e.g., 0.87) |
| `layerScores` | Per-layer scores and rationales (L1, L2, L3) |
| `judgeRunAt` | Timestamp of evaluation run |

- Filter for fine-tuning: `confidenceScore > 0.8`
- Analyze recurring weaknesses across layers
- Track quality trends over time

## Algorithms and Scoring Details

### Layer 1: Structural Validation

- Schema validator: Zod/TypeBox or custom TypeScript guards for required fields and types
- Strict mode: disallow unknown fields for deterministic checks

### Layer 2: Tool Correctness

- F1 score: precision/recall on expected vs. actual tool sets (case-insensitive canonical names)
- Sequence score: `LCS(actual, expected) / len(expected)`; penalize swaps/omissions

### Layer 3: Expert AI Review

- Prompt includes ground-truth facts, tool logs, and user intent; asks for scoring and rationale with concise citations to provided facts
- Guardrails: refuse speculative medical advice beyond provided evidence; flag uncertainty

## CI/CD Integration

Fast PR gate runs Layer 1 and 2 on changed areas with deterministic scores. Nightly runs all layers over curated fixtures and a rotating sample of real conversations.

```yaml
# .github/workflows/run-eval-tests.yml
name: Run Eval (PR)
on:
  pull_request:
    paths:
      - "judge/**"
      - "server/**"
jobs:
  evaluate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20 }
      - run: npm ci
      - run: npm run test:eval -- --layers L1,L2 --report junit
```

## Operational Runbook

<Tip>
Best practices to keep scores trustworthy and useful:

- Keep fixtures close to product scenarios; update with each feature change
- Alert on sudden composite deltas (>10%) between nightly runs
- Sample real conversations for Layer 3 to control costs while catching regressions
</Tip>

## FAQ

<AccordionGroup>
  <Accordion title="How is the composite score calculated when a layer is missing?">
    Weights are renormalized across present layers. In the PR gate, L1 and L2 are re-weighted to 30% and 70% respectively.
  </Accordion>
  <Accordion title="Can we trace a low composite back to root causes?">
    - **L1 failures:** schema errors or missing required fields
    - **L2 penalties:** missing tools or mis-ordered invocation (low LCS)
    - **L3 issues:** factual mismatches, unclear recommendations, poor tone
  </Accordion>
  <Accordion title="What threshold should we use for training data selection?">
    Start with `confidenceScore >= 0.8`. Periodically validate with spot checks and adjust per distribution shifts.
  </Accordion>
</AccordionGroup>
