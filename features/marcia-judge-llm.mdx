### 1. What is Judge LLM?

**Judge LLM** (LLM-as-a-Judge) uses a capable language model to automatically assess the quality, correctness, reasoning soundness, safety, and helpfulness of outputs generated by other AI systems.

Traditional automatic metrics (BLEU, ROUGE, exact match) only capture surface-level overlap or strict string matching. They fail on open-ended generation, multi-step reasoning, or semantically valid but differently phrased answers. Human evaluation provides the gold standard but is slow, expensive, inconsistent across reviewers, and non-scalable for thousands of daily outputs.

Judge LLM bridges this gap: it approximates human judgment at scale, applies consistent criteria, and produces explainable scores with reasoning traces. Comprehensive surveys (2024-2025, e.g., arXiv:2411.15594) and production benchmarks show well-designed judges achieve 80-90% human agreement often matching or exceeding typical inter-human agreement on subjective tasks.

### 3. Recommended Approach: Hybrid Evaluation
**Hybrid: Prometheus 2 (primary) + frontier fallback**

**Why hybrid?**

- Pure frontier-only (e.g., Claude Opus 4.6 / GPT-5 series) delivers highest raw reasoning judgment but incurs high API cost and latency at scale.
- Pure open-source-only risks missing edge cases or lacking generalization.
- Hybrid achieves near-frontier quality for 95%+ of evaluations at low cost, reserving expensive calls for calibration or ambiguous cases.
**Why Prometheus 2 as primary?** ([arXiv:2405.01535](https://arxiv.org/abs/2405.01535), still leading open judge in 2026 comparisons)

- Fine-tuned specifically for evaluation tasks → strong rubric adherence and fine-grained scoring
- Achieves ~0.85-0.9 Pearson correlation with human judgments on reasoning benchmarks (RewardBench, MT-Bench, FLASK)
- Open weights → fully self-hosted, no ongoing API dependency
- MoE 8x7B variant → efficient inference with strong performance
**Fallback trigger:** Low confidence from primary, out-of-distribution domain, or periodic meta-evaluation. Use frontier models (Claude Sonnet/Opus 4.6 class or GPT-4o) via existing infrastructure.

### 5. Why Pairwise + Position Swap?

LLMs show strong **position bias**, preferring the first (or sometimes last) presented response (60%+ in some studies, Wang et al. 2023; consistent across frontier models per 2024-2025 surveys).

**Solution:** Always run comparison twice (A then B, then B then A). Aggregate:

- Agreement → high-confidence winner
- Disagreement → detect bias, default to tie
This reduces position bias by ~40% and improves reliability (Prometheus paper, arXiv surveys).

[Image: image-20260320-112513.png]

### 6. Why Require Chain-of-Thought in Every Evaluation?

Direct scoring (“Rate 1-5”) often yields shallow, inconsistent outputs.

Requiring step-by-step analysis first (“Analyze logical flow, premises, inferences, then score”) forces the judge to explicitly justify its decision → improves consistency by ~25% and debuggability (Evidently AI guide, multiple benchmarks).

### 7. Key Biases & Mitigations
| Bias | Description | Mitigation |
| --- | --- | --- |
| Position | Favors first/last position | Mandatory swap + aggregation |
| Verbosity | Longer responses preferred | Rubric explicitly values conciseness |
| Self-enhancement | Favors own model’s style | Never self-judge; use separate judge model |
| Sycophancy | Agrees with assumed preference | Blind eval (no user metadata) |
| Authority/confidence | Rewards confident tone over correctness | Rubric: “Confidence ≠ evidence” |
| Format | Prefers bullets/headers | Format-agnostic criteria |

### 8. Model Choices

**Primary:** Prometheus 2 8x7B (MoE) ,  production judge (self-hosted via vLLM/Ollama) **Fallback:** Claude Sonnet 4.5 / Opus 4.6 or GPT-4o-class, strongest reasoning judges when needed

Frontier models excel at deep, nuanced reasoning dissection (e.g., Claude Opus 4.6 leads 2026 reasoning leaderboards), but Prometheus closes the gap for most use cases at far lower cost.

### 9. Rubric Design Principles

Good rubrics are:

- Specific and observable (“Contains clear cause-effect links” vs. “is logical”)
- Graduated with clear differences between scores
- Include negative examples for low scores
- Domain-aligned (veterinary safety differs from general chat)
- Evidence-based (judge must quote text)

### 10. Meta-Evaluation: Judging the Judge
A misaligned judge creates false confidence.

Maintain 50-100 expert-labeled calibration examples. Track:

- Human agreement rate (target ≥85%)
- Bias rates (≤15%)
- Score distribution & consistency
Re-calibrate quarterly or on drift.

### 11. When NOT to Use Judge LLM

- Pure factual verification → use retrieval/database check
- Code execution correctness → run actual tests
- Real-time blocking paths → async monitoring only
- Extremely low-stakes content → sampling sufficient
- Known adversarial inputs → human review
