# Marcia AI - Complete Architecture Documentation

**Author:** hamza
**Source:** Originally drafted in the personal Confluence space (now sunset); republished here as the current source of truth. New revisions live only in this Mintlify page.

**Marcia** is a multi-channel AI-powered veterinary assistant embedded throughout the CrestPetSystem platform. It provides intelligent conversational AI capabilities across four communication channels, leveraging an MCP-tool surface and a set of specialised subagents to help pet owners and veterinary clinics with:

- Pet health management and assessments
- Appointment booking and clinic discovery
- Document analysis (health records, insurance, lab results)
- AI-powered scans (poop analysis, tooth scans)
- Product recommendations and shopping
- Treatment plan tracking
- Network subscription management

## Web Portal Flow

```
Browser (Marcia widget)
        │
        ▼
Next.js Marcia page (client/src/app/marcia/)
        │
        ▼
NestJS API: /portals/marcia/chatbot/*
        │
        ├── Auth Guard (Better Auth session OR Turnstile public token)
        ├── Tenant resolver (subdomain → tenantId, optional)
        ▼
ChatbotService
        │
        ├── Persists message → ChatbotMessage / ChatbotConversations
        ├── (file upload) → S3 AGENT_CONVERSATION_IMAGES bucket
        ▼
LLM Router (server/src/app/llm-router/)
        │
        ├── MCP tool surface (appointment booking, scans, RAG, etc.)
        ├── Provider resolution (Anthropic primary, fallbacks)
        ▼
Response → SSE stream → browser
```

## SMS Flow

```
Pet owner SMS
        │
        ▼
Twilio webhook → POST /portals/marcia/sms/webhook
        │
        ├── Twilio signature validation (TwilioWebhookAuthGuard)
        ├── Phone-number → User lookup (or registration prompt)
        ▼
MarciaSmsService
        │
        ├── Persists message → SmsConversation / SmsMessage
        ├── (MMS) → S3 AGENT_CONVERSATION_IMAGES bucket
        ▼
LLM Router (same as web chat)
        │
        ▼
Outbound SMS via Twilio API (chunked to <=160 chars where possible)
```

## WhatsApp Flow

```
Pet owner WhatsApp message
        │
        ▼
Twilio Business API webhook → POST /portals/marcia/whatsapp/webhook
        │
        ├── TwilioWebhookAuthGuard (signature validation)
        ├── Conversation routing (existing thread vs. new)
        ▼
WhatsApp service
        │
        ├── WhatsApp message drafts → marcia-whatsapp.prisma models
        ├── Medical updates / proactive outreach (in progress)
        ▼
LLM Router (same as web chat)
        │
        ▼
Outbound WhatsApp via Twilio Business API
```

## Cross-Channel Concerns

- **Tenant context:** Marcia is a consumer-facing assistant. SMS, WhatsApp, and voice channels have no inherent tenant context — see the [S3 Tenant Isolation Audit](/operations/s3-tenant-isolation) "Chatbot & SMS Files" section for the explicit user-scoped vs. tenant-scoped decision.
- **Auth:** Crest uses [Better Auth](/guides/authentication) (1.4.22) for portal sessions. Webhook channels (Twilio, ElevenLabs) use signature-validation guards in `portals/marcia/<channel>/guards/`.
- **Storage:** Conversation media (images, audio) lives in the `AGENT_CONVERSATION_IMAGES` S3 bucket under `${userId}/${conversationId}/...` keys.
