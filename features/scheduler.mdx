### Appointment Booking and Scheduling System for Veterinary Clinics

Our system is designed to serve two primary user groups: **consumers** (pet parents) and **clinics** (veterinary hospitals). Each side interacts with the platform differently, based on their needs and roles within the appointment lifecycle.

---

#### Consumer Side - Pet Parent Experience
Consumers are the pet owners who wish to book appointments for their pets at participating clinics. They can initiate bookings through two main pathways:

**Standard Booking Access:**
The pet parent can browse and book services that the clinic has marked as publicly bookable. These may include general consultations, vaccinations, grooming, or any other service the clinic allows for self-scheduling.

**Authorized Workflow Access:**
Alternatively, the clinic may provide authenticated or invitation-based access, enabling consumers to follow a specific set of steps customized and approved for their pet. This can include multi-step procedures or follow-ups recommended by the clinic.

In both cases, the consumer selects the **desired service** and the **pet profile**, after which the system displays the available time slots. These are presented in a daily view across a rolling **two-week window**. Once the consumer selects a preferred time slot, they can **confirm the appointment**, which will then appear on their **Appointments page** with a **pending status**. The clinic will then review and approve the booking through their portal.

---

#### Clinic Side - Scheduling and Management Interface

The clinic side of the platform provides a robust and centralized **Master Board** for managing all aspects of the appointment schedule. The interface includes:

- **Room and Schedule Overview:**
  All clinic rooms are displayed along with their associated schedules and available controls. Staff can view appointments and procedural steps across days and rooms, enabling them to track the day's workflow at a glance.
- **Appointment Management:**
  Clinics can create new appointments for patients or edit existing ones. In both scenarios, the staff can define a sequence of **procedural steps**, such as diagnostics, imaging, or treatment stages. The system intelligently identifies the **optimal time slots** for each step, while also offering manual scheduling options if the clinic wishes to override the suggestion.
- **Status Updates and Configuration Controls:**
  Clinics can update appointment statuses (e.g., pending, confirmed, completed) and modify additional configurations related to procedures, timing, and resources.

---

#### Advanced Features and Emergency Handling

- **Emergency Booking Support:**
  In urgent cases, the clinic can mark a procedure as **high-priority**. The system will then identify and recommend the **earliest possible slot**, even if this requires **rescheduling** lower-priority appointments. Clinics are given options to review and approve the proposed changes before they are finalized.
- **Room Downtime Management:**
  If a particular room or equipment (e.g., X-ray machine) is temporarily unavailable, clinics can **disable** that room for a defined time period. The system automatically identifies appointments affected by this downtime and presents **rebooking options** using the next best available slots.

---

#### Authorization and Customization

Clinics have the ability to generate **authorization codes** that allow specific consumers to access certain procedures or workflows. They can also specify whether all procedural steps must be booked **on the same day** or can be distributed over multiple days.

---

#### Staff Utilization and Operational Insights

The platform tracks **staff availability** and **utilization** on a per-day basis. Clinics can view:

- Which staff members are on shift
- How their time is allocated across rooms and procedures
- Aggregated data that contributes to the clinic’s **overall utilization metrics**

These insights support better workforce planning and improve clinic efficiency.

---

#### Management of Procedures and Resources

Within the **Management page**, clinic administrators can:

- Create and manage **procedures** (i.e., bookable services)
- Assign **buffer durations** and apply **color codes** for visual clarity
- Allocate specific **rooms and staff** to each procedure
- These configurations are then used by the scheduling algorithm to optimize resource allocation during appointment booking.

# Scheduler Technical Documentation

## Appointment Booking and Scheduling System for Veterinary Clinics

### Overview

Our scheduling system serves two primary user groups: consumers (pet parents) and clinics (veterinary hospitals), providing an intelligent, high-performance appointment booking algorithm that optimizes resource allocation in real-time.

## Technical Implementation

### Core Algorithm: Matrix-Based Greedy Scheduling

Instead of using computationally expensive NP-complete algorithms, our system employs a **greedy matrix traversal algorithm** that efficiently finds optimal appointment slots by treating the scheduling problem as a 2D grid search.

#### Time Discretization Model

The system discretizes time into manageable blocks using 5-minute intervals. Each day is divided into 288 discrete time slots (24 hours × 12 slots per hour), creating a fine-grained scheduling grid that balances precision with computational efficiency. This discretization allows the system to handle complex scheduling requirements while maintaining sub-second response times.

#### Matrix Architecture

The scheduling engine constructs a 2D availability matrix where:

- **Rows represent resources**: Each row corresponds to either a staff member or a room
- **Columns represent time slots**: Each column represents a 5-minute time block
- **Cell values indicate availability**: Boolean values show whether a resource is available at a specific time

The matrix structure includes metadata columns at the beginning of each row to identify the entity type (STAFF or ROOM) and the specific entity ID, followed by 288+ boolean values representing availability across the scheduling window.

#### Algorithm Flow: The "Plane Sweep" Approach

**Phase 1: Data Preparation and Aggregation**
The system begins by executing a single, optimized database query that fetches all required scheduling data through SQL UNION operations. This includes staff availability windows, existing appointments, room status information, service step requirements, and any blocking restrictions. This approach dramatically reduces database round trips compared to multiple individual queries.

**Phase 2: Matrix Construction**
For each day in the scheduling window (typically 1-14 days), the algorithm:

**Staff Availability Processing**: For each staff member, creates an availability array initialized to false, then marks slots as available based on their scheduled work windows

**Appointment Conflict Resolution**: Removes slots that are already occupied by existing appointments, considering buffer times between procedures

**Room Status Integration**: Similarly processes room availability, accounting for maintenance periods, equipment downtime, and existing bookings

**Virtual Appointment Integration**: Incorporates any pending virtual appointments that may affect resource availability

**Phase 3: Greedy Plane Sweep Algorithm**
The core scheduling algorithm employs a "plane sweep" technique that efficiently traverses the availability matrix:

**Linear Traversal**: A virtual "plane" sweeps left-to-right through the time matrix, examining each time slot sequentially

**Availability Detection**: At each slot, the algorithm quickly checks if both required staff and room resources are available

**Contiguous Block Identification**: When availability begins, the algorithm marks the start of a potential booking window. When availability ends, it evaluates whether the duration meets the minimum service requirements

**Resource Assignment**: For valid availability blocks, the algorithm identifies which specific staff members and rooms are available for the entire duration

**Early Termination**: If only the first available slot is needed, the algorithm terminates immediately upon finding a valid option

### Key Performance Optimizations

#### 1. Single Query Data Fetching

- **Problem**: Traditional approaches make multiple database round trips
- **Solution**: One optimized SQL query with UNION ALL fetches all data types
- **Benefit**: Reduces database load from O(n) to O(1) queries

#### 2. Matrix-Based State Management

- **Problem**: Checking availability for each slot individually is O(n²)
- **Solution**: Pre-computed availability matrix enables O(n) traversal
- **Benefit**: Linear time complexity for availability checking

#### 3. Early Termination Strategies

The algorithm implements intelligent stopping conditions:

- Immediately returns when finding the first available slot if that's all that's needed
- Breaks out of inner loops the moment both required staff and room resources are found
- Avoids unnecessary computation once scheduling requirements are satisfied

#### 4. Efficient Block Detection

Instead of checking every possible time window which would be O(n²), the algorithm:

- Tracks contiguous availability blocks as it sweeps through time
- Only validates blocks that meet minimum duration requirements
- Achieves O(n) time complexity for block identification through single-pass traversal

### Master Board Architecture

**Real-time Resource Dashboard:**

All rooms displayed with live availability status

Staff schedules synchronized across the clinic network

### Intelligent Step Scheduling

The system optimizes multi-step appointment sequences by analyzing dependencies between procedural steps. For each step in a sequence, the algorithm calculates which previous steps must be completed first, then finds the optimal time slot that satisfies all dependency requirements while minimizing gaps between related procedures.

### Emergency Priority System

#### Priority-Based Slot Allocation

The system implements a four-tier priority hierarchy:

**IMMEDIATE**: Can override any lower priority appointment

**URGENT**: Can override SEMI_URGENT and ELECTIVE appointments

**SEMI_URGENT**: Can override ELECTIVE appointments only

**ELECTIVE**: Standard priority, cannot override others

**Conflict Resolution Algorithm:**

**Identify Conflicts**: When IMMEDIATE priority appointment requested

**Calculate Impact**: Determine which appointments can be displaced

**Suggest Alternatives**: Present rebooking options for affected appointments

**Atomic Transaction**: All changes committed together or none at all

**Tenant-Isolation Locking Strategy:**

**Per-Clinic Lock Isolation**: Each clinic operates with completely independent locking mechanisms, ensuring that high booking activity at one clinic doesn't block operations at other clinics

**Queue-Based Lock Management**: Booking requests for the same clinic are queued and processed sequentially using Promise-based coordination

**Timeout Protection**: 30-second timeout mechanism prevents indefinite lock holds due to system failures or network issues

**Automatic Lock Cleanup**: Background process continuously monitors and releases expired locks to prevent deadlock scenarios

**Lock Lifecycle Management:**

**Lock Acquisition**: When a booking request arrives, the system checks for existing locks and either acquires immediately or waits for the current lock to be released

**Critical Section Protection**: The entire booking process (availability check, conflict resolution, database writes) occurs within the protected critical section

**Graceful Release**: Locks are automatically released upon completion, error, or timeout, ensuring system availability

**Status Monitoring**: Real-time lock status tracking provides visibility into wait times and potential bottlenecks

**Race Condition Prevention:**

**Atomic Availability Checks**: Availability verification and booking confirmation happen atomically within the same database transaction

**Optimistic Concurrency Control**: The system uses version-based conflict detection for appointment modifications

**Rollback Mechanisms**: Failed bookings trigger automatic rollback of any partial changes

**Barrier Synchronization**: Multi-step appointment booking uses barrier synchronization to ensure all steps are validated before committing any step

### Performance Characteristics

#### Time Complexity

**Availability Checking**: O(d × s × r) where:

d = number of days (typically 1-14)

s = slots per day (288)

r = number of resources (staff + rooms)

**Block Finding**: O(n) linear sweep through availability matrix

**Conflict Detection**: O(k) where k = number of conflicting appointments

#### Space Complexity

**Matrix Storage**: O(d × s × r) for availability state

**Memory Footprint**: ~50KB per clinic for 14-day window with 20 resources

#### Database Performance

**Single Query Strategy**: Reduces database round trips by 90%

**Query Execution Time**: \<200ms for most clinic sizes

**Connection Pool Efficiency**: Supports 1000+ concurrent users per clinic

## Advanced Features

### Room Downtime Management

The system provides intelligent handling of equipment and room unavailability:

**Impact Analysis**: Automatically identifies all appointments affected by room downtime

**Alternative Slot Discovery**: Finds the next best available time slots for displaced appointments

**Comprehensive Rebooking**: Presents clinic staff with complete impact analysis and rebooking options

**Temporal Flexibility**: Supports both permanent room deactivation and temporary downtime periods

**Cascade Management**: Handles multi-step appointments where one step's room becomes unavailable

### Multi-Day Appointment Sequencing

The system supports complex procedures requiring multiple days with sophisticated dependency management:

**Step Dependencies**: Each procedure step can specify which previous steps must be completed first

**Time Gap Constraints**: Minimum and maximum gap requirements between related steps

**Cross-Day Optimization**: Intelligent scheduling across multiple days while respecting medical protocols

**Flexible Sequencing**: Support for both rigid sequential steps and flexible parallel steps

**Protocol Compliance**: Ensures medical procedures follow proper timing requirements
