## ✅ Validation Patterns

### String Validations

```
// Required field
name: z.string().min(1, "Name is required")

// Email
email: z.string().email("Invalid email address")

// Phone
phone: z.string().regex(/^\+?[1-9]\d{1,14}$/, "Invalid phone number")

// URL
website: z.string().url("Invalid URL").optional()
```

### Number Validations

```
// Range
age: z.number().min(0).max(150)

// Integer only
quantity: z.number().int("Must be a whole number")

// Positive
price: z.number().positive("Must be greater than 0")
```

### Selection Validations

```
// Enum
status: z.enum(["active", "inactive", "pending"])

// Boolean required true
acceptTerms: z.literal(true, {
  errorMap: () => ({ message: "You must accept the terms" })
})
```

### Complex Validations

```
// Conditional required
schema.refine(
  (data) => data.hasInsurance ? !!data.policyNumber : true,
  { message: "Policy number required", path: ["policyNumber"] }
)

// Cross-field validation
schema.refine(
  (data) => data.password === data.confirmPassword,
  { message: "Passwords don't match", path: ["confirmPassword"] }
)
```

Add asterisk to required field labels:

```

```

### Optional Fields

Mark optional fields explicitly:

```

```

Show counter for limited fields:

