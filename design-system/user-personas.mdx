# Crest Pet System - User Personas

> **Source Branch:** `prestaging`
> **Last Updated:** December 2024

<Note>
This is a UX design artifact with fictional personas. It describes intended user types and their needs, not a technical specification. Useful as design context but not a code reference.
</Note>

This document defines the primary user personas for each portal in the Crest Pet System.

---

## 🏥 Clinic Portal Users

### Persona 1: Dr. Sarah Chen - Veterinarian

**Demographics:**

- Age: 35-50
- Role: Lead Veterinarian / Practice Owner
- Tech Comfort: High
- Device: Desktop (primary), Tablet (exam room)


**Goals:**

- Review patient history quickly between appointments
- Document findings efficiently during exams
- Monitor practice performance and revenue
- Manage staff scheduling and workload


**Pain Points:**

- Time pressure between appointments (avg 15-30 min slots)
- Interruptions during documentation
- Difficulty finding historical records
- Complex multi-step procedures need coordination


**Key Workflows:**

1. View daily schedule → Navigate to patient → Review history → Document notes
2. Review lab results → Update diagnosis → Create treatment plan
3. Check practice metrics → Identify issues → Adjust staffing


**Design Implications:**

- Dense information displays are acceptable
- Keyboard shortcuts essential
- Fast load times critical
- Split-view layouts helpful


---

### Persona 2: Maria Rodriguez - Veterinary Technician

**Demographics:**

- Age: 25-40
- Role: Vet Tech / Nurse
- Tech Comfort: Medium-High
- Device: Tablet (primary), Workstation (secondary)


**Goals:**

- Prepare patients for appointments efficiently
- Record vitals and pre-exam notes
- Manage inventory and supplies
- Coordinate room assignments


**Pain Points:**

- Hands often occupied (holding animals)
- Needs large touch targets for tablet use
- Frequently interrupted
- Multiple tasks in parallel


**Design Implications:**

- Large, touch-friendly buttons (44x44px minimum)
- Quick-action buttons prominent
- Status indicators visible at glance
- Confirmation dialogs for critical actions


---

### Persona 3: Jessica Park - Receptionist

**Demographics:**

- Age: 22-35
- Role: Front Desk / Client Services
- Tech Comfort: Medium
- Device: Desktop (primary)


**Goals:**

- Book appointments quickly over phone
- Check in/out patients efficiently
- Process payments accurately
- Handle client inquiries


**Pain Points:**

- Phone calls while using system
- Clients waiting at desk
- Payment processing issues
- Schedule conflicts


**Design Implications:**

- Clear call-to-action buttons
- Minimal clicks for common tasks
- Visible calendar overview
- Payment flow streamlined

## 👨‍👩‍👧‍👦 Consumer Portal Users

### Persona 4: Emily & David Thompson - Pet Parents

**Demographics:**

- Age: 30-45
- Role: Pet Owner (dog, 2 cats)
- Tech Comfort: Medium
- Device: Mobile (70%), Desktop (30%)


**Goals:**

- Book vet appointments easily
- Understand pet health status
- Access medical records when needed
- Get answers to health questions


**Pain Points:**

- Anxiety about pet health
- Confusing medical terminology
- Difficulty remembering appointment details
- Overwhelmed by too many options


**Design Implications:**

- Simple, friendly language (no jargon)
- Guided workflows (wizards)
- Clear visual hierarchy
- Emotional support in UI copy
- Mobile-first design
- Large text for readability

## 🎯 Design Priorities by Portal

### Clinic Portal

1. **Speed** - Minimize clicks, keyboard shortcuts
2. **Density** - Information-rich displays acceptable
3. **Reliability** - Never lose data mid-entry
4. **Flexibility** - Support various workflows

### Consumer Portal

1. **Simplicity** - Guided experiences, clear language
2. **Reassurance** - Calming design, helpful messaging
3. **Mobile** - Mobile-first, touch-optimized
4. **Accessibility** - Large text, high contrast

### Enterprise Portal

1. **Overview** - Dashboard-centric design
2. **Drill-down** - Navigate from summary to detail
3. **Comparison** - Side-by-side views
4. **Export** - Reports and data export

### Platform Portal

1. **Power** - Full access, bulk operations
2. **Search** - Find anything quickly
3. **Audit** - Track all changes
4. **Efficiency** - Batch operations

### Referral Portal

1. **Clarity** - Clear status and next steps
2. **Communication** - Easy document sharing
3. **Tracking** - Follow referral lifecycle
4. **Coordination** - Connect parties efficiently
