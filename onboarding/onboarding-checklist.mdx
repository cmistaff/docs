Use this checklist when onboarding a new developer. Each tool requires an invite or access grant. Work through the list in order. Questions should be directed to **Bryant Parchinski** or emailed to [help@creststaff.com](mailto:help@creststaff.com).

<Note>
This is an **admin-facing** checklist. The new developer does not need to perform these steps themselves. The onboarding admin sends all invites and provisions access.
</Note>

---

## Tool Provisioning Checklist

<Steps>
  <Step title="Vercel">
    Send team invite from the Vercel dashboard.
  </Step>
  <Step title="Loom">
    Send workspace invite from Loom admin settings.
  </Step>
  <Step title="Supabase">
    Add to the Supabase organization.
  </Step>
  <Step title="Cursor">
    Send invite from Cursor team settings.
  </Step>
  <Step title="Slack">
    Send workspace invite via Slack admin panel.
  </Step>
  <Step title="GitHub Repos">
    Add to the `cmistaff` GitHub organization and grant access to relevant repositories.
  </Step>
  <Step title="Jira">
    Add to the `cammaratainc.atlassian.net` Jira instance.
  </Step>
  <Step title="Confluence">
    Add to the `cammaratainc.atlassian.net` Confluence workspace.
  </Step>
  <Step title="Twilio">
    Grant access to the Twilio account as needed.
  </Step>
  <Step title="AWS">
    Create IAM user or grant role-based access in the appropriate AWS account.
  </Step>
  <Step title="Claude Code (Anthropic Teams)">
    Add via claude.ai Organization Settings. Select the **Team** account, not API.
  </Step>
  <Step title="ChatGPT Codex (Team Plan)">
    Log in with the developer's `@creststaff.com` Google account.
  </Step>
  <Step title="Marcia Code API Key">
    Request an API key from Michael.
  </Step>
  <Step title="ElevenLabs">
    Send invite from the ElevenLabs workspace.
  </Step>
  <Step title="Tavus">
    Add to the Tavus account.
  </Step>
  <Step title="Hubstaff">
    Send invite from the Hubstaff organization (Crest Advisory Group, org ID 673357).
  </Step>
  <Step title="Stripe (if needed)">
    Grant dashboard access only if the developer's role requires it.
  </Step>
  <Step title="QStash">
    Provision access to QStash as needed.
  </Step>
  <Step title="Mistral">
    Add to the Mistral workspace.
  </Step>
  <Step title="Accounting Google Form">
    Share the accounting Google Form link with the new developer.
  </Step>
  <Step title="Otter (for past meetings)">
    Grant access to Otter.ai for meeting transcript history.
  </Step>
</Steps>

<Tip>
For a complete step-by-step onboarding workflow (including dev workstation setup, Tailscale VPN, and SentinelOne EDR), see the [Dev Workstation Setup](/onboarding/devbox-setup) guide.
</Tip>
