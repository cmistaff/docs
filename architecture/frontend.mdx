# Frontend Architecture

The CrestPet frontend is a Next.js 15 application using the App Router, React 19, TypeScript 5.8 (strict mode), and Tailwind CSS 4. It serves seven portals through subdomain-based routing. Luxon is the mandated datetime library (no native Date, moment, or dayjs).

## API Integration

All API communication uses auto-generated Orval hooks and functions. Direct `fetch` or `axios` calls are prohibited.

### Client-Side Rendering (CSR)

Use Orval-generated `useGet*` hooks in `"use client"` components:

```typescript
"use client";

import { useGetClinicAppointments } from "@/common/api/openapi";

export function AppointmentsList() {
  const { data, isLoading } = useGetClinicAppointments({
    page: 1,
    limit: 20,
  });

  if (isLoading) return <Spinner />;
  return <AppointmentTable appointments={data.items} />;
}
```

### Server-Side Rendering (SSR)

Use Orval-generated `get*` functions in server components and pass data as `initialData`:

```typescript
import { getClinicAppointments } from "@/common/api/openapi";

export default async function AppointmentsPage() {
  const data = await getClinicAppointments({ page: 1, limit: 20 });
  return <AppointmentsList initialData={data} />;
}
```

### Mutations

Use `useCreate*` / `useUpdate*` hooks and invalidate queries on success using Orval query keys:

```typescript
const { mutate } = useCreateClinicAppointment({
  mutation: {
    onSuccess: () => {
      queryClient.invalidateQueries({
        queryKey: getGetClinicAppointmentsQueryKey(),
      });
    },
  },
});
```

## State Management

| Data Type | Tool | Usage |
|-----------|------|-------|
| Server data | React Query | All data fetched from the API |
| Mutations | React Query | Create/update/delete operations |
| UI state | Redux Toolkit | Modals, filters, sidebar state |
| Forms | React Hook Form + Zod | Form validation and submission |
| URL state | Next.js router | Page parameters, search filters |

## Component Patterns

### Server Components by Default

Components are server components unless they need client-side interactivity. Add `"use client"` only for:
- `useState`, `useEffect`, or other React hooks
- Event handlers (onClick, onChange)
- Browser APIs (localStorage, window)
- Third-party client-only libraries

### Timezone Handling

All dates from the API are in UTC. The frontend converts to clinic-local time for display:

```typescript
import { toClinicTime } from "@/lib/utils/to-clinic-time.util";
import { fromClinicTimeToUtc } from "@/lib/utils/from-clinic-time-to-utc.util";
import { useClinicTimezone } from "@/hooks/use-clinic-timezone";

// Display: UTC → clinic local
const displayTime = toClinicTime(appointment.startAt, clinicTimezone);

// Send to API: clinic local → UTC
const utcTime = fromClinicTimeToUtc(localDateTime);
```

### RBAC Permission Checks

Every clinic dashboard section enforces permission checks:

```typescript
import { useHasPermission } from "@/hooks/useHasPermission";
import { useHasWriteAccess } from "@/hooks/useHasWriteAccess";
import { withPermission } from "@/lib/hoc/withPermission";
import { Permission, PermissionAction } from "@prisma/client";

// Hook-based check (array of typed permission objects)
const canViewBilling = useHasPermission([
  { permission: Permission.CLINIC__PAYMENTS, actions: [PermissionAction.READ] },
]);
const canEditBilling = useHasWriteAccess(Permission.CLINIC__PAYMENTS);

// HOC-based route protection (two arguments, not curried)
export default withPermission(AppointmentsPage, Permission.CLINIC__APPOINTMENTS);
```

### Forms

All forms use Zod schemas with React Hook Form:

```typescript
import { useForm } from "react-hook-form";
import { createZodResolver } from "@/lib/utils/create-zod-resolver";
import { appointmentSchema } from "./schemas/appointment.schema";

const form = useForm({
  resolver: createZodResolver(appointmentSchema),
  defaultValues: { ... },
});
```

## Design System

The UI follows a "SpaceX meets Apple" design philosophy using shadcn/ui components, with:

- Clean spacing and minimal visual noise
- Mobile-first responsive layouts
- WCAG 2.1 AA accessibility compliance
- Tailwind CSS 4 for all styling
- Consistent dark blue (#1E3A5F) brand color

## Code Generation

After any backend API changes, regenerate the frontend API client:

```bash
pnpm api:generate
```

This runs `pnpm api:schema` (generates OpenAPI spec from NestJS decorators) followed by `pnpm api:client` (Orval generates React Query hooks and TypeScript types).

All types in `@/common/api/openapi/models` are auto-generated. Never create frontend type definitions for API data manually.
