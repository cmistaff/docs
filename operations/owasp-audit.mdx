## Why OWASP API Security?

OWASP API Security Top 10 is the industry standard for REST API threat modelling. It complements our existing perimeter defences (Cloudflare Access, Turnstile, AWS WAF) by auditing the **application layer**, where most breaches originate.

### Critical and High Findings

#### 1. No Rate Limiting on Auth Endpoints (API2 + API4)

**Gap:** Better Auth login/signup endpoints bypass NestJS `ThrottlerModule` entirely. Password reset has no `@Throttle` decorator. No account lockout mechanism exists.

**Impact:** An attacker can brute-force credentials or flood password reset requests indefinitely.

```bash
# No rate limit - attacker can send thousands of requests
POST /api/auth/sign-in/email  { email: "victim@clinic.com", password: "attempt1" }
POST /api/auth/sign-in/email  { email: "victim@clinic.com", password: "attempt2" }
... (unlimited attempts, no lockout)
```

**Files:** `server/src/modules/app/app.module.ts`, `server/src/modules/portals/shared/auth/password-reset.controller.ts`

---

#### 2. AI-Generated SQL Injection (API10)

**Gap:** The insights module executes LLM-generated SQL via `$queryRawUnsafe` with no parameterisation, no read-only enforcement, and no connection role restriction.

**Impact:** A compromised MCP secret or prompt injection attack allows arbitrary SQL: `UPDATE`, `DELETE`, `DROP TABLE`.

```typescript
// server/src/modules/portals/clinic/insights/services/natural-language/insights-sql-query.service.ts:67
async executeSqlQuery(sql: string): Promise<string> {
  const limitedSql = this.enforceRowLimit(sql); // regex only - no parameterisation
  const result = await this.prismaService.$queryRawUnsafe(limitedSql); // ARBITRARY SQL
}
```

#### 3. No HTTP Security Headers (API8)

**Gap:** Helmet.js is not installed. Neither NestJS nor Next.js sets CSP, X-Frame-Options, HSTS, X-Content-Type-Options, or Referrer-Policy.

**Impact:** The application is vulnerable to clickjacking (iframe embedding), MIME-type sniffing attacks, and has no Content Security Policy to mitigate XSS.

```
# Response headers (current)
HTTP/1.1 200 OK
Content-Type: application/json
                              <- No X-Frame-Options
                              <- No Content-Security-Policy
                              <- No Strict-Transport-Security
                              <- No X-Content-Type-Options
```

**Files:** `server/src/main.ts`, `client/next.config.ts`

---

#### 4. File Upload Memory Exhaustion (API4)

**Gap:** `FileUploadInterceptor` has no multer `fileSize` limit. The 10MB JSON body parser limit does not apply to multipart uploads. File size is validated only after the entire file is buffered in memory.

**Impact:** An attacker uploads a multi-GB file to any of 11+ upload endpoints. The server buffers the entire file in memory before the application-level `validateFile()` check fires, causing OOM and service disruption.

```bash
# Attacker sends 5GB file - server buffers it all before rejecting
POST /clinics/patients/123/documents  Content-Type: multipart/form-data
Body: [5GB file]  ->  Server OOM before validateFile() runs
```

**File:** `server/src/decorators/file-upload/file-upload-interceptor.decorator.ts`

---

#### 5. SSRF via AI Image URL Fetching (API7)

**Gap:** AI chatbot extracts image URLs from user messages and fetches them server-side via `getBase64File()` with no domain allowlist or private IP blocking.

**Impact:** A crafted message containing `http://169.254.169.254/latest/meta-data/` triggers the server to fetch AWS instance metadata (IAM credentials, tokens).

```
User message: "Check this image http://169.254.169.254/latest/meta-data/iam/security-credentials/"
-> Server fetches AWS metadata endpoint
-> IAM credentials returned to AI context
```

**Files:** `server/src/modules/common/utils/get-base64-file.utils.ts`, `server/src/modules/portals/shared/ai/utils/image-resolver.util.ts`

---

#### 6. CORS Regex Not Anchored (API8)

**Gap:** `new RegExp("localhost:3000")` is used for CORS origin validation with `credentials: true`. The unanchored regex matches substrings.

**Impact:** An attacker's domain `evilsite-localhost:3000.attacker.com` passes CORS validation, enabling cross-origin cookie theft.

```typescript
// server/src/main.ts:78
app.enableCors({
  origin: new RegExp(commonConfig.app.corsRegexp), // "localhost:3000" matches substring
  credentials: true, // cookies sent cross-origin
});

// Attacker's domain passes:
"https://evilsite-localhost:3000.attacker.com".match(/localhost:3000/) // matches
```

#### 7. ValidationPipe Missing Whitelist (API8)

**Gap:** The global `ValidationPipe` does not set `whitelist: true` or `forbidNonWhitelisted: true`. Undeclared properties in request bodies pass through to handlers silently.

**Impact:** Mass assignment. An attacker injects fields not in the DTO (e.g., `isAdmin: true`, `tenantId: "other-tenant"`) that may be picked up by downstream processing.

```typescript
// POST /clinics/patients - DTO declares { name, email }
// Attacker sends:
{ "name": "John", "email": "john@test.com", "roleId": "admin-role-uuid" }
// "roleId" passes through ValidationPipe unchecked
```

**File:** `server/src/config/app-pipes.config.ts`

---

#### 8. Password Reset Open Redirect (API2)

**Gap:** The `baseUrl` field in password reset accepts any HTTPS URL. It is validated as a URL but not restricted to application domains.

**Impact:** Attacker submits a reset request with `baseUrl: "https://attacker.com/phish"`. The victim receives a legitimate-looking reset email pointing to the attacker's domain.

```typescript
// POST /auth/password-reset/request
{ "email": "victim@clinic.com", "baseUrl": "https://attacker.com/fake-reset" }
// -> Victim gets email: "Click here to reset: https://attacker.com/fake-reset?token=abc123"
```

**File:** `server/src/modules/portals/shared/auth/dto/requests/request-password-reset.dto.ts`
