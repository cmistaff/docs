## Executive Summary

The Crest Pet System codebase was audited against the Shai-Hulud 2.0 npm supply chain attack that compromised ~700 packages between November 21-27, 2025.

**Result:** All package versions in the project are **SAFE**. While some package names appear in the dependency tree, the specific versions installed are **NOT** the compromised versions.

## Packages Checked

- **Total compromised packages in attack:** 795
- **Packages found in project:** 4

## Why We're Safe

The Shai-Hulud 2.0 attack targeted **specific versions** of npm packages published between November 21-24, 2025. The Crest Pet System uses:

1. **Locked dependencies** via `pnpm-lock.yaml`
2. **Older versions** that predate the attack
3. **Exact version pinning** in package.json


This means even though package names match, the actual code installed is from **safe, pre-attack versions**.

## Attack Details (Reference)

According to Wiz Security Research:

- **Attack window:** November 21-24, 2025
- **Packages compromised:** ~700 packages
- **Attack method:** Maintainer account compromise
- **Payload:** Credential exfiltration (GitHub tokens, AWS keys, GCP credentials)
- **Impact:** 25,000+ repositories affected with stolen secrets published to GitHub

### What the malware did:

- Exfiltrated environment variables (AWS_ACCESS_KEY_ID, GITHUB_TOKEN, etc.)
- Scraped `.git/config` files
- Uploaded stolen data to attacker-controlled GitHub repositories
- Executed during `preinstall` phase (very broad attack surface)
