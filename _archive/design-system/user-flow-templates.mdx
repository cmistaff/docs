## 📋 Flow Template Structure

### Header Information

```
Flow Name: [Name]
Portal: [Clinic/Consumer/Enterprise/Platform/Referral]
Actor: [User persona]
Trigger: [What initiates the flow]
Success Outcome: [End state]
Version: [X.X]
Last Updated: [Date]
```

### Required Sections

1. **Entry Points** - How users access this flow
2. **Main Flow** - Primary happy path
3. **Decision Branches** - Alternative paths
4. **Error Handling** - Error scenarios
5. **Exit Points** - How users leave the flow

## 📝 Example Flow: Book Appointment

### Header

```
Flow Name: Book an Appointment
Portal: Consumer
Actor: Pet Owner (Emily Thompson persona)
Trigger: User clicks "Book Appointment" from dashboard or pet profile
Success Outcome: Appointment confirmed with confirmation displayed
Version: 1.0
Last Updated: December 2024
```

### Flow Steps

**Step 1: Entry**

- Dashboard → "Book Appointment" button
- Pet Profile → "Schedule Visit" button
- Navigation → Appointments → "New Appointment"

**Step 2: Select Pet**

- If 1 pet: Auto-select, skip to Step 3
- If multiple pets: Show pet selector
- If no pets: Redirect to "Add Pet" flow

**Step 3: Select Clinic**

- Show nearby clinics (location-based)
- Show previously visited clinics
- Search by name/zip

**Step 4: Select Service Type**

- Wellness Exam
- Sick Visit
- Vaccination
- Follow-up
- Other

**Step 5: Select Date/Time**

- Calendar view (month)
- Available slots highlighted
- Next available shown prominently

**Step 6: Confirm Details**

- Summary card with all selections
- Add notes field (optional)
- Terms acceptance checkbox

**Step 7: Confirmation**

- Success message
- Appointment details
- Add to calendar option
- View appointment button

### Error States

| Error | Handling |
| --- | --- |
| No availability | Show next available date, offer waitlist |
| Network error | Show retry button, preserve selections |
| Session expired | Redirect to login, preserve state if possible |

### Loading States

| Step | Loading UI |
| --- | --- |
| Clinics | Skeleton list |
| Calendar | Skeleton grid |
| Confirmation | Button spinner |

### Mobile Considerations

- Full-screen steps (wizard pattern)
- Bottom-anchored "Continue" button
- Swipe gestures for calendar

### Accessibility Notes

- Keyboard navigable calendar
- Screen reader announcements for selection changes
- Focus management between steps
