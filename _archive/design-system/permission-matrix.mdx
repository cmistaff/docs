# Permission Matrix (RBAC)

> **Source Branch:** `prestaging`
> **Last Updated:** December 2024

This document defines all permissions in the system and their mapping to navigation/features.

---

## 🔐 Permission System Overview

### Architecture

Permissions flow: **Tenant → Tenant Role → Role Permissions → Tenant User**

- Each tenant has roles (e.g., "Veterinarian", "Receptionist")
- Each role has a set of permissions with access levels (READ or WRITE)
- Users are assigned roles and inherit those permissions

### Access Levels

| Level | Description |
| --- | --- |
| READ | View-only access to the feature |
| WRITE | Full access including create/update/delete |

## 👨‍👩‍👧‍👦 Consumer Portal Permissions

The Consumer Portal uses **plan-based feature gating** rather than role-based permissions. All authenticated users have access to base features; premium features are controlled by plan subscription.

See **Plan Features** document for details.

---

## 🛡️ Platform Portal Permissions

Platform portal access is controlled at the user level:

| Role | Description |
| --- | --- |
| SUPER_ADMIN | Full system access |
| ADMIN | Standard admin access |
| SUPPORT | Read-only with support tools |

---

## 🔗 Referral Portal Permissions

Referral portal uses simplified access control:

- Authenticated users from partner clinics have full access
- Access is tenant-scoped (can only see own referrals)

## 🎨 UI Implementation

### Visual Indicators

| Scenario | UI Treatment |
| --- | --- |
| No permission | Hide navigation item entirely |
| Plan-gated (no plan) | Show disabled, with tooltip |
| READ-only access | Show, disable edit/delete buttons |
| WRITE access | Full functionality |
