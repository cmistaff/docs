# Plan Features & Feature Gating

> **Source Branch:** `prestaging`
> **Last Updated:** December 2024

This document defines all plan-gated features across tenant types and how they affect the UI.

---

## 🎯 Feature Gating Overview

### Concept

Plan features control **what functionality is available** based on a tenant's subscription plan. Unlike permissions (which control WHO can access), plan features control WHETHER a feature exists at all for the tenant.

### Dual Gating

Some features require BOTH:

1. **Permission** - User must have the right role
2. **Plan Feature** - Tenant must have the feature in their plan

## 🎨 UI Implementation

### Key Design Principles

1. **Never Hide Completely** - Gated features should be visible but disabled, not hidden. This shows users what's possible and creates upgrade motivation.
2. **Clear Messaging** - Always provide a tooltip explaining what the feature does, why it's unavailable, and how to get access.
3. **Graceful Degradation**:Clinic features: Assume enabled (fail-open)
4. Consumer features: Assume disabled (fail-closed)
5. **Loading States** - While feature data loads, show navigation items in loading state. Don't flash enabled→disabled.
